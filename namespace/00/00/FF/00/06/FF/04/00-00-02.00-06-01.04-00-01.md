```
node_id: 00-00-02.00-06-01.04-00-01
node_nam: Node Identifier Segment
node_tag: NODE_SEG
```

## Rules

- `FF` is syntactically valid in a [NODE_ID] string but MUST NOT appear in an assigned [NODE_ID].
- A position MAY constrain its assignable [NODE_SEG] values to a controlled vocabulary.
- For an unconstrained position, the authority governing the parent scope assigns non-zero values. Assignments are unique within one parent scope at that position and MAY be sparse.
- Once a non-zero [NODE_SEG] value is assigned within one parent scope at one position, it MUST NOT be reassigned within that same scope to identify a different [FOLIO].

## Constraints

- Each [NODE_SEG] is two or more hexadecimal digits. [NODE_SEG] width is fixed at [FROID_NS] instantiation; all [NODE_SEG] within the namespace share this width.
- Values `01` to `FE` are assignable unless the position document further constrains them.
- `00` is the identity value at any position and is not an assignable non-identity value.
- `FF` is reserved and must not appear in any assigned [NODE_ID].
- Only the all-Fs value at the [FROID_NS] canonical [NODE_SEG] width is reserved. Intermediate-width variants (e.g., `00FF` when the namespace width is four digits) are assignable and do not conflict with the reserved value.

## Relations

- [NODE_SEG] is a component of [NODE_ID].
- Each [NODE_SEG] corresponds to exactly one position.
- [NODE_SEG] values are concatenated with dot separators to form [NODE_ID].

## Notes

- Controlled-vocabulary position such as MM and TT use their declared assignable values rather than per-scope assignment.
