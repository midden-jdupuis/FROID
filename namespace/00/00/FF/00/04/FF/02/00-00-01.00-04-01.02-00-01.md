## Definition

[MODE_2] is the governance and description mode. It holds normative and descriptive knowledge that governs or explains [MODE_1] objects and their relationships.

A [MODE_2] artifact defines what is allowed, what is required, what something means, and how it relates to other objects. It does not define the object itself and does not implement or render it.

[MODE_2] is a distributed rule and interpretation system. A single [MODE_2] artifact may apply to many [FOLIO] simultaneously at branch, variant, or component scope.

## Artifact Types

[MODE_2] uses both types:
- **[TYP_M2T1]** — Mode 2 Type 1 Markdown (alternative; for human-readable governance)
- **[TYP_M2T2]** — Mode 2 Type 2 YAML (canonical; machine-readable constraints)

Type 2 (YAML) is the canonical choice for governance rules and constraints — machine-readable and composable.

## Rules

- [MODE_2] MUST NOT define the object itself — that belongs in [MODE_1].
- [MODE_2] MUST NOT implement or render — that belongs in [MODE_3].
- [MODE_2] MUST NOT contain raw data instances.
- A [MODE_2] artifact may govern multiple [FOLIO] via scope selectors (branch-level, variant-level, component-level).
- [MODE_2] artifacts state what is allowed, required, forbidden, or means.
- [MODE_2] applies to [FOLIO] via overlay scope selectors with MM=02.

## Constraints

- A [MODE_2] artifact must answer at least one of: "Is this valid?", "Why does this exist?", "What replaces it?", or "What rules apply to it?".
- A [MODE_2] artifact must not duplicate the declarative content of the [MODE_1] it governs.
- [MODE_2] governing a [MODE_1] MUST NOT redefine or override the [MODE_1]'s semantic; it MUST constrain or explain.
- [MODE_2] Type pairing: Both Type 1 (Markdown) and Type 2 (YAML) are available; Type 2 is canonical.

## Relations

- [MODE_2] is a child of [MODE] at MM=02.
- [MODE_2] governs [MODE_1] objects.
- [MODE_2] constrains [MODE_3] realizations.
- [MODE_2] applies to [FOLIO] via overlay scope selectors (binding=overlay, MM=02).
- Multiple [MODE_2] artifacts may govern the same [MODE_1] (via different selectors).
- [MODE_2] may reference other [MODE_2] artifacts for rule composition.
