```
node_id: 00-00-01.00-01-01.00-00-01
node_nam: Node Identifier
node_tag: NODE_ID
```

## Definition

[NODE_ID] is the nine-segment readable identifier for a position in [FROID_NS].

[NODE_ID] is composed of nine two-digit hexadecimal [SEGMENT] values in fixed order: RR.VV.MM.BB.CC.TT.UU.KK.II.

The canonical form of a [NODE_ID] is nine two-digit uppercase hexadecimal [SEGMENT] separated by dots.

A syntactically valid [NODE_ID] denotes one position in [FROID_NS]. Assignment is separate from syntactic validity.

## Rules

- A [NODE_ID] consists of exactly nine [SEGMENT] values in fixed order: RR-VV-MM.BB-CC-TT.UU-KK-II.
- A [NODE_ID] is canonical in uppercase; parsers MUST normalize lowercase input to canonical uppercase form.
- Two [NODE_ID] are equal iff their canonical forms are byte-identical.
- Only an assigned [NODE_ID] identifies a [FOLIO]; syntactic validity does not create assignment.
- An unassigned [NODE_ID] denotes an empty namespace position with no [FOLIO].
- When lower [SEGMENT] values are `00`, the assigned [NODE_ID] identifies the [FOLIO] at the depth of its deepest non-zero [SEGMENT].
- A [NODE_ID] can be brought into alignment with [X660] by the deterministic forward mapping: convert each hex segment to decimal and prepend `1.3.6.1.4.1`.

## Constraints

- Each [NODE_ID] consists of exactly nine [SEGMENT], each exactly two hexadecimal digits.
- `00` in a [SEGMENT] denotes the identity position at that position and is not an assignable non-identity value.
- `FF` is reserved: syntactically valid in a [NODE_ID] string but MUST NOT appear in any assigned [NODE_ID].
- Values `01` to `FE` are assignable unless the position document further constrains them.
- [NODE_ID] is immutable once assigned; changes require creating new [FOLIO] with supersession relations.

## Relations

- [NODE_ID] is the primary identifier for [FOLIO] in [FROID_NS].
- [NODE_ID] is composed of nine [SEGMENT] in fixed order corresponding to nine position levels.
- Each [SEGMENT] in a [NODE_ID] corresponds to exactly one position.
- [NODE_ID] identifies zero or one [FOLIO] when assigned (uniqueness enforced globally).
- [NODE_ID] has a human-readable name [NODE_NAM] and a short tag [NODE_TAG].
- [NODE_ID] targets are referenced by [FOLIO] via [FLIO_PYLD] payload lists and [REFRENCE] links.
