```
node_id: 00-00-01.00-03-01.02-00-00
node_nam: Folio Artifact
node_tag: FLIO_ART
```

## Definition

[FLIO_ART] is any payload member of a [FOLIO] other than its [FLIO_REF].

An [FLIO_ART] file shares the [NODE_ID] of its parent [FOLIO] in the [SEG_GRAN] (granular) segments. The [SEG_BINN] (bin) segments MM and TT of the [FLIO_ART]'s [NODE_ID] specify the artifact's mode and type. The [FLIO_ART] filename is the [FLIO_ART]'s [NODE_ID] concatenated with the file extension corresponding to TT.

[FLIO_ART] files reside in the same leaf directory as the parent [FOLIO]'s [FLIO_REF].

## Rules

- An [FLIO_ART] is an immutable file; once committed it MUST NOT change.
- An [FLIO_ART] is enumerated in the parent [FLIO_REF] via [FLIO_PLD].
- One [FLIO_ART] per (mode, type) coordinate per parent [FOLIO]. Multiple [FLIO_ART] at the same (mode, type) coordinate are not permitted.

## Constraints

- An [FLIO_ART] filename MUST be `{NODE_ID}.{ext}` where `{NODE_ID}` is the artifact's [NODE_ID] and `{ext}` corresponds to its TT.
- An [FLIO_ART] [NODE_ID] MUST share the parent [FOLIO]'s [SEG_GRAN] segments exactly. Only the [SEG_BINN] segments may differ.
- An [FLIO_ART] MUST NOT use the `.frd` extension; that is reserved for [FLIO_REF].

## Relations

- [FLIO_ART] is associated with one parent [FOLIO].
- [FLIO_ART] is referenced by one [FLIO_REF] via [FLIO_PLD].
- [FLIO_ART] type is classified by [TYPE].
- [FLIO_ART] mode is classified by [MODE].
