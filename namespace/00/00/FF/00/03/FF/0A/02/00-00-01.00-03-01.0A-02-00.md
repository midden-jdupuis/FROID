## Definition

[FLO_SPDN] is the downstream supersession direction in [FLIO_SUP].

The manifest field name is `flo_spdn`. Its value is a list of [NODE_ID] that supersede this [FOLIO] — the [FOLIO] that replace this one.

Each entry resolves to another [FOLIO] in [FROID_NS]. Validation is reference-resolution.

## Rules

- `flo_spdn`, when present, is a list of [NODE_ID].
- Each entry MUST resolve to an existing [FOLIO].

## Constraints

- Duplicate [NODE_ID] in `flo_spdn` are not permitted.
- A [NODE_ID] in `flo_spdn` MUST NOT appear in this [FOLIO]'s `flo_spup`.

## Relations

- [FLO_SPDN] is the downstream half of the [FLIO_SUP] field-pair.
- [FLO_SPDN] is reciprocal with [FLO_SPUP]: if A.`flo_spup` contains B, then B.`flo_spdn` MUST contain A.
