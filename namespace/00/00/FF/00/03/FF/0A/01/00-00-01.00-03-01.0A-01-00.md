## Definition

[FLO_SPUP] is the upstream supersession direction in [FLIO_SUP].

The manifest field name is `flo_spup`. Its value is a list of [NODE_ID] that this [FOLIO] supersedes — the [FOLIO] this one replaces.

Each entry resolves to another [FOLIO] in [FROID_NS]. Validation is reference-resolution.

## Rules

- `flo_spup`, when present, is a list of [NODE_ID].
- Each entry MUST resolve to an existing [FOLIO].
- The list is ordered only as a record of attachment; ordering does not imply precedence among superseded [FOLIO].

## Constraints

- Duplicate [NODE_ID] in `flo_spup` are not permitted.
- A [NODE_ID] in `flo_spup` MUST NOT appear in this [FOLIO]'s `flo_spdn`.

## Relations

- [FLO_SPUP] is the upstream half of the [FLIO_SUP] field-pair.
- [FLO_SPUP] is reciprocal with [FLO_SPDN]: if A.`flo_spup` contains B, then B.`flo_spdn` MUST contain A.
