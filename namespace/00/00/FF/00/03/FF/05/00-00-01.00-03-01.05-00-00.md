```
node_id: 00-00-01.00-03-01.05-00-00
node_nam: Folio Payload
node_tag: FLIO_PLD
```

## Definition

[FLIO_PLD] is the ordered list of [FLIO_ART] references declared within an [FLIO_REF] manifest.

Each [FLIO_PLD] entry is a string identifying one [FLIO_ART] file present in the same leaf directory as the [FLIO_REF]. The string is the [FLIO_ART] filename: the [NODE_ID] of the [FLIO_ART] concatenated with its file extension.

## Rules

- A [FLIO_PLD] entry MUST be the filename of a file present in the [FOLIO]'s leaf directory.
- A [FLIO_PLD] entry MUST NOT equal `{NODE_ID}.frd` for the parent [FOLIO]; the [FLIO_REF] is not a member of its own payload.
- The [FLIO_PLD] list MAY grow as additional [FLIO_ART] are attached over time.
- Existing entries in [FLIO_PLD] MUST NOT be modified or removed.

## Constraints

- Order in [FLIO_PLD] is significant only as a record of attachment sequence; it does not imply precedence.
- Duplicate filenames in [FLIO_PLD] are not permitted.

## Relations

- [FLIO_PLD] is recorded by [FLIO_REF].
- [FLIO_PLD] entries reference [FLIO_ART] files.
