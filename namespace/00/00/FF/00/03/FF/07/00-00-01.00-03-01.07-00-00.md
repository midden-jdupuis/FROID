## Definition

[FLIO_HSH] is the hash field set in [FLIO_REF]. Each hash field carries one [FLO_HASH] value and stores a SHA-256 digest covering a defined portion of the manifest or its lineage.

Hash fields in [FLIO_REF]:
- `record_hash` — carries [FL_HSCOR]
- `payload_hash` — carries [FL_HSPLD]
- `supersession_hash` — carries [FL_HSSPR]

Each hash field is a 64-character lowercase hexadecimal string. The fields are computed at distinct lifecycle moments and serve distinct integrity purposes.

## Rules

- Each [FLIO_HSH] field's name is fixed and the field's value class is one [FLO_HASH] member.
- A [FLIO_HSH] value is computed at the moment specified for its [FLO_HASH] kind and is immutable thereafter.

## Constraints

- The set of [FLIO_HSH] fields is closed at this revision: `record_hash`, `payload_hash`, `supersession_hash`.
- Each field, when present, MUST be a 64-character lowercase hexadecimal SHA-256 digest.

## Relations

- [FLIO_HSH] values are enumerated by [FLO_HASH].
- [FLIO_HSH] fields are recorded in [FLIO_REF].
- [FLIO_HSH] applies to one [FOLIO].
