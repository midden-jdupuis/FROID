```
node_id: 00-00-01.00-05-01.03-00-00
node_nam: Artifact File Header Block
node_tag: TYPE_HED
```

## Definition

[TYPE_HED] is a structured header block placed at the top of an [FLIO_ART] file that encodes the [FLIO_ART]'s identity within the file content itself.

[TYPE_HED] carries the [FLIO_ART]'s identity fields as in-file content, distinct from the [FLIO_REF] manifest that describes the parent [FOLIO]. [TYPE_HED] provides self-identification: a reader of an [FLIO_ART] file in isolation can determine the [NODE_ID], [NODE_TAG], and ownership of the [FLIO_ART] without consulting the [FLIO_REF].

The fields [TYPE_HED] carries: `node_id`, `node_nam`, `node_tag`. Additional fields MAY include `created` and `owner`. The binding of the parent [FOLIO] is recorded in [FLIO_REF] and is not duplicated in [TYPE_HED].

The encoding of [TYPE_HED] depends on the [FLIO_ART]'s [TYPE]:

- For [TYP_MKDN]: YAML frontmatter delimited by `---` lines at the top of the file, immediately followed by the markdown body.
- For [TYP_YAML]: top-of-file YAML key/value pairs, followed by a blank line and the artifact's structured content.
- For [TYP_CSV]: comma-separated key/value rows preceding the data table, with the keys and values placed in a reserved column range that does not overlap the table columns.

## Rules

- [TYPE_HED]'s `node_id` MUST equal the [NODE_ID] encoded in the [FLIO_ART]'s filename.
- [TYPE_HED]'s `node_tag` MUST be the canonical [NODE_TAG] for the [NODE_ID].
- [TYPE_HED] precedes any other [FLIO_ART] content in the file.
- [TYPE_HED] is required for every [FLIO_ART] whose [TYPE] admits a header block.

## Constraints

- The set of valid [TYPE_HED] fields is closed at this revision: `node_id`, `node_nam`, `node_tag`, `created`, `owner`.
- `node_id` is required; other fields are optional.
- The encoding format is dictated by the [FLIO_ART]'s [TYPE]; [TYPE_HED] does not impose a single serialization.

## Relations

- [TYPE_HED] is a structural property of [FLIO_ART].
- [TYPE_HED] is a member of [TYPE].
- [TYPE_HED] is encoded differently per [TYPE], but carries the same fields across encodings.
- [TYPE_HED] mirrors a subset of the fields recorded in [FLIO_REF].
